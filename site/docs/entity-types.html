Entity Types Reference

Info

Entity types determine how VectorScaleDB compresses and indexes your data. Each type has optimized compression sensitivity for its data characteristics. Use the type that best matches your data source.

Core Types

Type Name Description
GENERIC Default type for unclassified temporal vectors

Usage Example

Specify the entity type when ingesting data to enable optimized compression and indexing:

JSON 
{
  "entity_id": "sensor-42",
  "entity_type": "LINK_METRIC",
  "vector": [0.1, 0.8, 0.3, 0.9],
  "timestamp": "2025-06-01T12:00:00Z"
}
Tip

If you don't specify an entity type, VectorScaleDB defaults to GENERIC. For best compression and query performance, always use the most specific type that matches your data source.
