Getting Started

What is VectorScaleDB?

VectorScaleDB is the first database where time-series data and vector embeddings share a unified index -- the Temporal-Semantic Tree. This architecture enables queries that are impossible in any existing database: trajectory similarity search, vector-aware temporal compression, behavioral regime detection, and scene differencing across time.

Instead of forcing you to stitch together a time-series database, a vector store, and a streaming engine, VectorScaleDB provides a single system purpose-built for temporal-semantic workloads.

Info

VectorScaleDB is designed for real-time entity tracking, anomaly detection, pattern matching, and behavioral intelligence across any domain with temporal-spatial data.

Key Capabilities

  • Temporal-Semantic Index -- HNSW per time-chunk with temporal continuity edges, sub-millisecond KNN across any time range, snapshot/restore
  • Vector-Aware Compression -- 100-1,000x compression via drift-based regime detection, WAL + crash recovery, adaptive threshold tuning
  • 4D Scene Intelligence -- Native 4D Gaussian Splatting storage, scene differencing, LOD streaming, anchor-relative offset storage
  • Trajectory Analysis -- FastDTW similarity, anomaly scoring, behavioral forecasting, change-point detection, DBSCAN semantic clustering
  • Enterprise Security -- AES-256-GCM envelope encryption, per-tenant key management, automated key rotation, RBAC, SHA-256 audit chains, GDPR deletion, differential privacy
  • Distributed Intelligence -- Jump consistent hashing, SWIM gossip protocol, CRDTs, scatter-gather coordination, entity migration, federated template exchange
  • Cross-Domain Intelligence -- Cascade detection, predictive forecasting, cross-domain alerting pipeline, DBSCAN domain composition analysis
  • Decentralized Architecture -- libp2p transport layer, IPFS segment archival, content-addressed CIDs, differential privacy on segments
  • Fleet-Scale Pattern Matching -- Behavioral pattern detection across millions of entities, cross-node federated template discovery

Quick Start

1. Connect

Python 
from vectorscaledb import Client

client = Client("https://your-instance.vectorscaledb.com")
curl 
# Verify connectivity
curl https://your-instance.vectorscaledb.com/api/v1/health

2. Ingest Data

Python 
client.ingest(
    entity_id="sensor-42",
    vector=[0.1, 0.8, 0.3, 0.9],
    entity_type="LINK_METRIC"
)
curl 
curl -X POST https://your-instance.vectorscaledb.com/api/v1/ingest \
  -H "Content-Type: application/json" \
  -d '{
    "entity_id": "sensor-42",
    "vector": [0.1, 0.8, 0.3, 0.9],
    "entity_type": "LINK_METRIC"
  }'

3. Query

Python 
results = client.temporal_knn(
    query_vector=[0.1, 0.7, 0.4, 0.8],
    start_time="2025-01-01T00:00:00Z",
    end_time="2025-12-31T23:59:59Z",
    k=5
)
curl 
curl -X POST https://your-instance.vectorscaledb.com/api/v1/query/temporal-knn \
  -H "Content-Type: application/json" \
  -d '{
    "query_vector": [0.1, 0.7, 0.4, 0.8],
    "start_time": "2025-01-01T00:00:00Z",
    "end_time": "2025-12-31T23:59:59Z",
    "k": 5
  }'
Tip

Temporal KNN combines vector similarity with time-range filtering in a single index traversal -- no post-filtering required. This is what makes it faster than chaining a vector search with a time-series query.

Supported Protocols

  • REST API -- 80+ endpoints covering ingestion, querying, administration, and streaming
  • gRPC -- Protocol Buffer definitions for high-throughput, low-latency communication
  • GraphQL -- Queries, mutations, and subscriptions for flexible data access
  • SSE / WebSocket -- Server-Sent Events and WebSocket streams for real-time notifications
  • SDKs -- Native clients for Python, Rust, and Node.js with full type safety

Industry Adapters

VectorScaleDB includes adapters that translate domain-specific formats into temporal-semantic vectors on ingest:

Category Formats
Autonomous Vehicles MCAP, LAS/PCD, SPZ, glTF
Financial Markets FIX 4.4/5.0, CSV tick data, ITCH/OUCH
Networking & CDN NetFlow/sFlow/IPFIX, SNMP, gNMI, Prometheus
Bio-Computing NWB, HDF5, NEV/NSx, SpikeInterface, MEA streams
Info

Adapters handle format parsing, vector extraction, and entity type mapping automatically. You send raw domain data; VectorScaleDB indexes it as temporal-semantic vectors.
