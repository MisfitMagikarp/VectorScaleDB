Infrastructure

Networking & CDN Intelligence

Transform network telemetry into multi-dimensional behavioral vectors. Detect anomalies against learned baselines, optimize routes with multi-factor quality weighting, and track topology changes across your entire infrastructure.

Multi-Dimensional Link Metric Vectors

Every network link becomes a multi-dimensional temporal vector, capturing the full behavioral fingerprint of the connection at every measurement interval.

Behavioral Vectors
Holistic Link Fingerprinting
Each network link is captured as a multi-dimensional behavioral vector covering key performance indicators -- utilization, latency, error rates, and more. These dimensions combine to form a behavioral fingerprint that distinguishes congestion from hardware failure, normal load from anomalous traffic, and healthy links from degrading ones.
Temporal
Behavioral Trajectory Over Time
Link vectors are tracked over time as temporal trajectories. VectorScaleDB's regime detection identifies when a link transitions between behavioral states: normal operation, congestion, degradation, or failure. Stable periods compress automatically while transitions retain full resolution for root-cause analysis.
Similarity
Cross-Link Comparison
Compare behavioral patterns across links using vector similarity. "Which links in region B are behaving like the link that failed in region A last week?" Similarity search across link trajectories enables proactive maintenance and failure prediction based on historical behavioral patterns.

Prometheus Adapter Integration

Pull metrics directly from Prometheus endpoints. VectorScaleDB maps Prometheus metric families into temporal vectors automatically.

Integration
Zero-Config Metric Mapping
Point VectorScaleDB at any Prometheus-compatible /metrics endpoint. Gauge and counter metrics are automatically vectorized: related metrics (e.g., interface_bytes_in, interface_bytes_out, interface_errors) are grouped into a single multi-dimensional temporal vector per entity. Labels become entity metadata for filtered queries.
Retention
Compressed Long-Term Retention
Prometheus retention is typically 15-30 days. VectorScaleDB ingests Prometheus data and compresses it through behavioral regime detection, retaining years of operational history in a fraction of the storage. Query last year's traffic patterns with the same latency as last hour's.

Route Optimization & Traffic Analysis

Multi-factor route quality scoring and traffic pattern analysis powered by temporal-semantic queries over link behavioral histories.

Routing
Multi-Factor Quality Weighting
Route optimization uses the full multi-dimensional link vector, not just latency. A route with low latency but rising queue depth and increasing jitter scores lower than a route with slightly higher latency and stable behavioral metrics. Weights are configurable per traffic class (voice, video, bulk).
Detection
Anomaly Detection with Baselines
VectorScaleDB learns the normal behavioral trajectory of every link. Anomalies are scored by deviation from the learned baseline, not static thresholds. A link at 90% utilization during business hours is normal; at 90% at 3 AM, it's a 0.95 anomaly. Baselines adapt automatically through regime detection.
Topology
Topology Change Detection
When network topology changes (link failures, reroutes, capacity upgrades), VectorScaleDB detects them as regime transitions in the affected links' behavioral trajectories. Each topology event is timestamped and annotated with the contributing links, enabling root-cause correlation.

CDN Edge Profiling

Profile every CDN edge node as a behavioral entity. Detect performance degradation, optimize cache placement, and correlate edge behavior with origin patterns.

Profiling
Edge Behavioral Fingerprints
Each CDN edge becomes a CdnEdge entity with temporal vectors capturing hit ratio, response latency, bandwidth utilization, cache fill rate, and error rate. Similarity search across edges identifies performance outliers: "which edges are behaving differently from their peers in the same region?"
Optimization
Cache Placement Intelligence
Trajectory analysis on content access patterns reveals which content should migrate between edges. VectorScaleDB's regime detection identifies when access patterns shift (viral content, regional events) and surfaces cache rebalancing opportunities before cache miss rates spike.

Supported Formats & Protocols

Flow Data
NetFlow / sFlow / IPFIX
Native ingestion of Cisco NetFlow v5/v9, sFlow v5, and IPFIX records. Flow records are vectorized by source/destination pair with dimensions for bytes, packets, duration, and TCP flags.
Monitoring
SNMP / gNMI
SNMP polling and gNMI streaming telemetry. Interface counters, CPU/memory utilization, and environmental sensors are mapped to temporal vectors per device.
Metrics
Prometheus
Direct scraping of Prometheus /metrics endpoints. Automatic metric grouping, label-based entity decomposition, and compressed long-term retention beyond Prometheus native limits.

Example Workflow

Ingest Prometheus link metrics and detect network anomalies with learned baselines.

# Configure Prometheus adapter for core router metrics
curl -X POST http://localhost:8080/v1/adapters/prometheus \
  -H "Content-Type: application/json" \
  -d '{
    "name": "core-routers",
    "endpoint": "http://prometheus:9090",
    "scrape_interval": "15s",
    "metric_mappings": [
      {
        "prometheus_metric_prefix": "interface_*",
        "entity_type": "LinkMetric",
        "entity_id_label": "interface"
      }
    ],
    "label_filters": {
      "device_role": "core-router"
    },
    "compression": {
      "enabled": true
    }
  }'

# Response:
# {
#   "adapter_id": "adp_prom_core_01",
#   "status": "active",
#   "entities_discovered": 847,
#   "scrape_interval": "15s",
#   "first_scrape_at": "2026-03-09T10:00:15Z"
# }
# Detect anomalies across all core router links using learned baselines
curl -X POST http://localhost:8080/v1/query/network-anomaly \
  -H "Content-Type: application/json" \
  -d '{
    "entity_type": "LinkMetric",
    "scope": {
      "adapter_id": "adp_prom_core_01",
      "time_range": {
        "start": "2026-03-09T09:00:00Z",
        "end": "2026-03-09T10:00:00Z"
      }
    },
    "baseline": {
      "method": "learned",
      "training_window": "7d",
      "time_of_day_aware": true
    },
    "threshold": 0.7,
    "include_contributing_dimensions": true,
    "include_topology_impact": true
  }'
{
  "anomalies": [
    {
      "entity_id": "link_core01_eth4_7",
      "device": "core-router-01",
      "interface": "Ethernet4/7",
      "anomaly_score": 0.92,
      "detected_at": "2026-03-09T09:34:12Z",
      "baseline_regime": "weekday-morning-normal",
      "contributing_dimensions": [
        { "dimension": "queue_depth", "deviation": 3.8 },
        { "dimension": "jitter", "deviation": 2.4 },
        { "dimension": "packet_loss", "deviation": 1.9 }
      ],
      "topology_impact": {
        "affected_routes": 23,
        "affected_prefixes": 1847,
        "cascade_risk": "medium"
      }
    }
  ],
  "total_links_scanned": 847,
  "anomalies_detected": 3,
  "latency_ms": 34.2
}
