Mobility

Autonomous Vehicles & Fleet Intelligence

Ingest entire MCAP recordings, voxelize LiDAR sweeps, align multi-modal sensors via DTW, and query similar driving trajectories across thousands of vehicles in a single temporal-semantic query.

MCAP Recording Ingestion

Native ingestion of ROS bag recordings in MCAP format. Every sensor channel becomes a temporal vector stream, automatically indexed by the Temporal-Semantic Tree.

Ingestion
Topic-Level Decomposition
MCAP files are decomposed by ROS topic. Each topic (/lidar_points, /camera/image_raw, /imu/data, /gps/fix) becomes an independent temporal vector stream with its own compression regime and index entry. No pre-processing pipelines required.
Metadata
Drive-Level Provenance
Every ingested recording retains full provenance: vehicle ID, drive session, route hash, weather conditions, firmware version. Query by any metadata dimension alongside temporal-semantic filters for precise scenario retrieval.
Scale
Fleet-Scale Batch Ingestion
Bulk-ingest terabytes of MCAP recordings from overnight fleet uploads. VectorScaleDB's WAL ensures crash-safe writes, and adaptive compression kicks in as each stream stabilizes, achieving 1000x ratios on steady-state sensor telemetry. Available as a managed cloud platform with multi-tenant fleet isolation and plan-based quotas.

LiDAR Point Cloud Voxelization

Raw point clouds become compact voxel occupancy vectors. Each LiDAR sweep is spatially discretized, compressed, and indexed for temporal-semantic queries.

Processing
Spatial Discretization
LAS and PCD point clouds are voxelized into configurable grid resolutions. Each voxel grid becomes a VoxelOccupancy entity with an occupancy vector suitable for similarity search. Compare scene structures across time, locations, and vehicles without point-by-point alignment.
Efficiency
Occupancy Compression
Voxel occupancy vectors compress dramatically through drift-based regime detection. Static environment regions (buildings, road surfaces) compress at 1000x+, while dynamic regions (intersections, construction zones) maintain finer temporal resolution. Storage costs drop from petabytes to terabytes.

Multi-Modal Sensor Alignment via DTW

Align LiDAR, camera, IMU, GPS, and radar streams that arrive at different rates and with different latencies using FastDTW temporal warping.

Alignment
Cross-Sensor Temporal Warping
LiDAR at 10Hz, cameras at 30Hz, IMU at 200Hz, GPS at 1Hz. FastDTW aligns these heterogeneous streams into a unified temporal reference frame. Query "what did all sensors observe at the moment the anomaly occurred" without manual timestamp interpolation.
Detection
Sensor Drift Detection
VectorScaleDB's change-point detection identifies when sensors fall out of calibration. A camera's embedding drift relative to LiDAR ground truth surfaces calibration degradation days before it manifests as a safety-critical perception failure.
Fusion
Fused Scene Representation
After alignment, sensor modalities merge into a fused SensorFrame entity. Each frame captures the full environmental state at a given instant: 3D geometry from LiDAR, texture from cameras, motion from IMU, position from GPS. One query, one fused answer.

Fleet-Scale Compression

1000x compression on sensor telemetry across thousands of vehicles. Store years of fleet data at the cost of days.

Behavioral Regimes
Drive State Compression
Highway cruising, urban stop-and-go, parking maneuvers, highway merging: each driving regime compresses into a single segment descriptor. A 2-hour highway drive that generates 720,000 sensor frames compresses to under 50 regime segments while preserving every behavioral transition.
Economics
Storage Cost Reduction
A fleet of 10,000 vehicles generating 50GB/day each produces 500TB of raw data daily. VectorScaleDB's vector-aware compression reduces this to under 500GB without losing the ability to query trajectory similarity, detect anomalies, or replay historical scenarios.

HD Map Maintenance via Scene Differencing

Every vehicle becomes a map sensor. Detect real-world changes by comparing current observations against the HD map baseline.

Detection
Change Detection Pipeline
VectorScaleDB's scene differencing compares voxelized LiDAR observations against stored MapElement baselines. New construction, lane reconfigurations, missing signage, and temporary obstacles are detected as semantic drifts in the spatial embedding space. Changes are localized to specific map tiles and timestamped.
Consensus
Fleet-Consensus Updates
A single vehicle's observation is noise. When 15 vehicles report the same spatial anomaly at the same location within 4 hours, VectorScaleDB's trajectory clustering (DBSCAN) confirms the change. Map updates propagate to the fleet with confidence scores derived from observation density and sensor agreement.

Supported Formats & Protocols

Recording
MCAP
Native MCAP ingestion with topic-level decomposition. Full support for ROS 1 and ROS 2 message types, including PointCloud2, Image, Imu, NavSatFix, and custom message definitions.
Point Clouds
LAS / PCD
ASPRS LAS 1.4 and PCL PCD formats for LiDAR point clouds. Automatic coordinate system detection, intensity normalization, and voxelization at configurable grid resolutions.
3D Scenes
SPZ / glTF
SPZ for Gaussian Splatting scene storage and glTF 2.0 for mesh-based scene representations. Scene differencing works natively across both formats for HD map and digital twin applications.
Entity Types
Native AV Entity Types
VoxelOccupancy TrackedObject MapElement SensorFrame

Example Workflow

Ingest an MCAP recording, then query for similar driving trajectories across the fleet.

# Ingest an MCAP recording from vehicle VH-0042
curl -X POST http://localhost:8080/v1/ingest/mcap \
  -H "Content-Type: application/json" \
  -d '{
    "source": "/recordings/2026-03-09/VH-0042-drive-session-14.mcap",
    "vehicle_id": "VH-0042",
    "session_id": "drive-session-14",
    "topics": [
      "/lidar/points",
      "/camera/front/image_raw",
      "/imu/data",
      "/gps/fix"
    ],
    "voxel_resolution": "auto",
    "compression": {
      "enabled": true
    },
    "metadata": {
      "route": "HQ-to-TestTrack-A",
      "weather": "clear",
      "firmware": "v4.2.1"
    }
  }'
