Spatial Computing

3D Spatial Computing & Digital Twins

Persist 4D Gaussian Splatting scenes with temporal evolution, detect spatial changes through scene differencing, stream level-of-detail representations for AR/VR, and manage digital twin lifecycles with IoT sensor fusion.

4D Gaussian Splatting Persistence

The first database with native 4DGS storage. Gaussians are not blobs in an object store. They are indexed, queryable, temporally evolving entities.

Static
GaussianStatic
Persistent environmental Gaussians: buildings, terrain, infrastructure. Ingested from SPZ or glTF captures, each Gaussian stores position (x, y, z), covariance matrix, spherical harmonic coefficients, and opacity. Static Gaussians compress at extreme ratios since they rarely change, making long-term scene archival efficient.
Dynamic
GaussianDynamic
Gaussians that evolve over time: vehicles, people, seasonal vegetation, construction progress. Each dynamic Gaussian has a temporal trajectory of position, covariance, and appearance parameters. VectorScaleDB's compression captures the behavioral regimes of spatial evolution: a tree's appearance compresses across seasons into four regime segments.
Transient
GaussianTransient
Short-lived Gaussians: weather effects, temporary objects, lighting changes. Transient Gaussians have explicit lifespans and are automatically pruned from active indices after expiration. Query "what transient objects appeared in this spatial region during the last 24 hours?" for security and monitoring use cases.

Scene Differencing for Change Detection

Compare scene snapshots across time. VectorScaleDB computes spatial diffs at the Gaussian level, identifying exactly what changed, where, and when.

Detection
Gaussian-Level Diff
Scene differencing operates on individual Gaussians: added Gaussians (new objects), removed Gaussians (demolished structures), modified Gaussians (appearance changes, position shifts). Each diff is localized in 3D space and timestamped. The diff itself is a queryable entity, enabling searches like "find all structural changes within 100m of this coordinate in the last month."
Applications
Change Detection Workflows
Construction progress monitoring: diff weekly scene captures to track building progress. Facility security: diff hourly captures to detect unauthorized modifications. Environmental monitoring: diff seasonal captures to track vegetation changes, erosion, and land use. Each workflow uses the same scene differencing API with different temporal resolution.

LOD Streaming for AR/VR

Deliver the right level of detail to each client based on distance, bandwidth, and device capability. VectorScaleDB manages the full LOD hierarchy.

Streaming
Bandwidth-Adaptive Delivery
LOD streaming sends coarse Gaussians first for immediate scene rendering, then progressively refines with higher-detail representations. VectorScaleDB manages LOD levels per spatial region, enabling clients to request "give me LOD-2 for everything within 50m and LOD-0 for everything beyond 200m" in a single query.
Spatial
View-Frustum Culling
Spatial indexing enables server-side view-frustum culling. Only Gaussians visible from the client's current position and orientation are transmitted. As the user moves, VectorScaleDB streams delta updates: newly visible regions at appropriate LOD, deactivation signals for out-of-view regions.
Temporal
Temporal LOD
Beyond spatial LOD, VectorScaleDB provides temporal LOD: request the scene state at any past timestamp with configurable temporal resolution. A facility manager reviewing last week's digital twin gets LOD-0 (daily snapshots). An incident investigator gets LOD-3 (per-minute resolution) for a specific 2-hour window.

Spatial Anchor Clustering & IoT Fusion

Cluster spatial anchors for persistent AR experiences and fuse IoT sensor data with spatial context for living digital twins.

Anchors
Persistent Spatial Anchors
AnchorPoint entities store 6DoF poses with uncertainty ellipsoids. DBSCAN clustering groups co-located anchors into stable reference frames. Anchor-relative offset storage ensures that AR content placed by one user appears correctly for another user, even when individual anchor detections vary. Anchor drift is detected through temporal trajectory analysis.
IoT Fusion
Sensor-Spatial Context Binding
Bind IoT sensor streams to spatial locations within the digital twin. A temperature sensor at coordinate (12.4, 3.1, 2.8) becomes a temporal vector stream anchored in 3D space. Query "show me all sensor readings within 5m of this Gaussian cluster" to correlate environmental conditions with spatial observations. VectorScaleDB unifies the spatial and temporal dimensions natively.

Digital Twin Lifecycle Management

From initial capture to decommissioning. VectorScaleDB manages the complete lifecycle of spatial digital twins.

Capture
Initial Scene Capture
Ingest the initial 4DGS scene from a LiDAR scan, photogrammetry pipeline, or NeRF-to-3DGS conversion. VectorScaleDB classifies each Gaussian as static, dynamic, or transient based on capture metadata and establishes the baseline scene state for future differencing.
Evolution
Continuous Update Pipeline
As the physical environment changes, incremental scene captures are diffed against the stored twin. Only changed Gaussians are updated. IoT sensors provide continuous telemetry. The digital twin evolves in real time, with full temporal history preserved through compressed regime segments.
Query
Temporal-Spatial Queries
Query the digital twin at any point in its lifecycle: "show me the facility as it was 6 months ago." "What changed on the 3rd floor between January and March?" "Which IoT sensors in Building A detected anomalous readings during the same period that construction was detected in the scene diff?" One database, one query language.

Supported Formats & Protocols

Gaussian Splatting
SPZ
Native SPZ format for compressed Gaussian Splatting scenes. Direct ingestion with automatic Gaussian classification (static/dynamic/transient) and spatial indexing. Sub-second scene loads for millions of Gaussians.
3D Standard
glTF 2.0
Industry-standard glTF format for mesh-based digital twins. Scene graph decomposition maps glTF nodes to VectorScaleDB entities with temporal tracking per node. Extensions supported for PBR materials and animation.
Point Clouds
LAS / PCD / E57
LiDAR point cloud formats for raw spatial capture. Automatic voxelization, coordinate system alignment, and conversion to Gaussian representations for unified scene management.
