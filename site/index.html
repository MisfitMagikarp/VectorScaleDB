The Multi-System Problem

Every organization with temporal-spatial data stitches together 3-5 systems. VectorScaleDB replaces them all.

Time-Series Databases
Can't do similarity
Great at temporal queries, but no concept of vector similarity. Cannot compare behavioral trajectories or detect semantic patterns across time.
Vector Databases
Can't handle time
Great at similarity search, but no native time dimension. Cannot answer "what changed over time?" or compress temporal patterns.
Current Reality
Everyone stitches together 3-5 systems
TimescaleDB + Milvus + custom ETL + application logic. Fragile pipelines, no unified queries, no behavioral intelligence.
The Solution
One engine for temporal-semantic queries
VectorScaleDB unifies time-series, vector embeddings, and 3D scene data in a single index. Trajectory similarity, behavioral compression, anomaly detection, and fleet-scale pattern matching — all native, all in one query.

Quick Start

Ingest data and query in seconds with any language.

from vectorscaledb import Client

client = Client("http://localhost:8080")

# Ingest a temporal vector
client.ingest(
    entity_id="sensor-42",
    vector=[0.1, 0.8, 0.3, 0.9],
    entity_type="LINK_METRIC"
)

# Temporal KNN — find similar vectors in a time range
results = client.temporal_knn(
    query_vector=[0.1, 0.7, 0.4, 0.8],
    start_time="2025-01-01T00:00:00Z",
    end_time="2025-12-31T23:59:59Z",
    k=5
)
use vectorscale_client::Client;

let client = Client::new("http://localhost:8080");

// Ingest a temporal vector
client.ingest("sensor-42", &[0.1, 0.8, 0.3, 0.9])
    .entity_type("LINK_METRIC")
    .send().await?;

// Temporal KNN — find similar vectors in a time range
let results = client.temporal_knn(&[0.1, 0.7, 0.4, 0.8])
    .time_range(start, end)
    .k(5)
    .send().await?;
# Ingest a temporal vector
curl -X POST http://localhost:8080/v1/ingest \
  -H "Content-Type: application/json" \
  -d '{
    "entity_id": "sensor-42",
    "vector": [0.1, 0.8, 0.3, 0.9],
    "entity_type": "LINK_METRIC"
  }'

# Temporal KNN — find similar vectors in a time range
curl -X POST http://localhost:8080/v1/query/temporal-knn \
  -H "Content-Type: application/json" \
  -d '{
    "query_vector": [0.1, 0.7, 0.4, 0.8],
    "start_time": "2025-01-01T00:00:00Z",
    "end_time": "2025-12-31T23:59:59Z",
    "k": 5
  }'
