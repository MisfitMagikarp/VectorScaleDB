The Two-Database Problem

Today you need two systems, stitched together in application code. VectorScaleDB replaces both.

Time-Series Databases

  • Great at temporal queries
  • No concept of vector similarity
  • Cannot compare behavioral trajectories
  • Compression ignores semantics

Vector Databases

  • Great at similarity search
  • No native time dimension
  • Cannot answer "what changed over time?"
  • No temporal compression

VectorScaleDB

  • Temporal + semantic queries natively
  • Trajectory similarity via DTW
  • Behavioral change-point detection
  • Vector-aware compression (100-1000x)

Key Innovations

Temporal-Semantic Tree (TST)

HNSW per time-chunk with temporal continuity edges. Neither a time index nor a vector index — both. Enables temporal KNN with sub-millisecond latency.

Vector-Aware Compression

Stores behavioral regimes, not individual data points. When an entity's behavior is stable, a single segment replaces thousands of vectors. 100-1000x compression beyond columnar.

Trajectory Queries

DTW-based trajectory similarity, behavioral change-point detection, anomaly scoring, forecasting, and DBSCAN clustering — all as native database operations.

Architecture

Rust engine with MongoDB storage. Purpose-built for temporal-semantic workloads.
